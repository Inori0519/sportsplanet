# 跃动星球：校园运动激励工具设计方案文档

> 文档版本：v1.0  
> 项目类型：游戏化运动激励网页工具  
> 目标形态：可通过公开链接访问的前端网页原型  
> 推荐技术栈：HTML + CSS + JavaScript + localStorage  
> 产品关键词：大学生运动、游戏化激励、社交共建、成就系统、ADHD 友好设计、校园地图、公会联赛  

---

## 1. 产品名称与定位

### 1.1 产品名称

**跃动星球**

### 1.2 副标题

**校园运动激励工具**

### 1.3 Slogan

> 把每一次运动，变成点亮校园星球的能量。

### 1.4 一句话定位

**跃动星球是一款面向大学生的游戏化运动激励网页工具，通过三分钟启动、校园地图探索、成就徽章、契约好友、公会共建、足迹占领和学院联赛机制，将运动从“自律任务”转化为可持续、可社交、可积累的校园冒险。**

### 1.5 产品核心价值

传统运动工具通常强调数据记录，例如步数、距离、时长和卡路里。但对大学生来说，真正的问题往往不是“不知道运动有益”，而是：

- 不想开始；
- 一个人运动太无聊；
- 没有即时反馈；
- 计划太宏大，执行太困难；
- 运动成果没有可展示、可积累的价值；
- 一旦中断，就很容易彻底放弃。

“跃动星球”的设计重点是将运动转化为一种**可反馈、可展示、可协作、可长期积累的游戏化行为**。

## 2. 项目背景

### 2.1 背景问题

大学生群体普遍存在运动频率不足和运动计划难以坚持的问题。很多学生虽然知道运动有益，但在真实生活中会受到课程压力、作息不规律、社交惰性、拖延、注意力分散和缺少同伴陪伴等因素影响，导致运动计划难以持续。

常见现象包括：

1. 收藏很多健身教程，但很少真正开始；
2. 想跑步，但想到 3 公里就失去动力；
3. 运动几天后中断，之后再也没有恢复；
4. 一个人运动缺少陪伴和外部反馈；
5. 运动成果只体现为冰冷数字，缺乏情绪价值；
6. 对 ADHD 或执行功能困难用户而言，启动、规划和持续执行更困难。

### 2.2 设计机会

如果将运动行为转化为游戏任务，并通过即时反馈、社交协作、成就积累和低门槛启动机制降低执行难度，就有机会提升用户运动频率和持续性。

因此，本项目提出“跃动星球”：

> 用户的每一次运动都会转化为“源力”，用于点亮校园地图、升级公会浮空岛、挑战世界 Boss、争夺校园区域控制权，并为学院联赛贡献能量。

## 3. 目标用户

### 3.1 核心用户

| 用户类型 | 典型特征 | 核心需求 |
|---|---|---|
| 普通大学生 | 想运动但缺少坚持动力 | 低门槛任务、即时奖励、成就反馈 |
| 宿舍/社团用户 | 有天然社交关系 | 团队挑战、公会共建、集体荣誉 |
| 运动新手 | 害怕强度过高 | 微任务、渐进式目标 |
| 拖延用户 | 总想“明天再开始” | 一键启动、短时任务、即时反馈 |
| ADHD 或注意力困难用户 | 难启动、难规划、易中断 | 三分钟启动、任务拆分、随机卡片、低压力重启 |
| 班级/学院组织者 | 希望提高整体运动参与度 | 学院联赛、班级榜单、团队激励 |

### 3.2 用户痛点

| 痛点 | 表现 | 产品解决方式 |
|---|---|---|
| 启动困难 | 想运动但迟迟不开始 | 三分钟启动舱 |
| 缺少即时反馈 | 运动后只有数据，没有成就感 | XP、源力、徽章、地图点亮 |
| 独自运动无聊 | 没有朋友一起，容易放弃 | 公会、契约好友、拓荒信号 |
| 计划难执行 | 目标太大，执行门槛高 | 微任务拆分、随机任务卡 |
| 容易中断 | 断签后觉得失败 | 低压力重启、恢复类成就 |
| 成果不可见 | 运动数据不能展示 | 成就墙、地图、周报、浮空岛 |
| 缺少集体荣誉 | 运动与班级/学院关系弱 | 学院圣杯、班级联赛 |

## 4. 设计原则

### 4.1 低启动门槛

产品不要求用户一开始完成高强度运动，而是鼓励用户先完成最小动作。

核心文案：

> 今天不需要完美运动，先动 3 分钟就算赢。

### 4.2 即时反馈

每次完成运动后，用户应立即获得可见奖励：

- XP 增加；
- 源力增加；
- 建造值增加；
- 地图区域点亮；
- 徽章解锁；
- 公会进度提升；
- Boss 血量下降。

### 4.3 轻压力社交

社交系统强调陪伴、协作和鼓励，而不是强排名羞辱。排行榜存在，但不应成为唯一动力。

### 4.4 不惩罚失败

用户中断后不清零、不羞辱、不强制惩罚，而是提供“重新开始”的奖励机制。

例如：

- 中断后重新运动可获得“重启勇者”徽章；
- 低能量日完成 3 分钟任务也可获得奖励；
- 公会目标未完成时进入“休整状态”，下周可继续挑战。

### 4.5 ADHD 友好

产品不提供医疗诊断或治疗，仅提供执行功能友好的交互设计。

设计重点包括：

- 一键开始；
- 微任务拆分；
- 随机任务卡；
- 倒计时；
- 即时奖励；
- 少选择、少负担；
- 可视化进度；
- 允许中断后恢复。

### 4.6 校园场景化

产品所有玩法尽量与校园生活绑定：

- 操场；
- 图书馆；
- 宿舍；
- 体育馆；
- 湖边小径；
- 社团；
- 班级；
- 学院。

## 5. 世界观设定

### 5.1 基础世界观

大学校园被抽象为一颗“校园星球”。这颗星球被迷雾、虚空裂隙和沉睡区域覆盖。用户作为“拓荒者”，通过现实运动产生“源力”，逐步点亮校园区域、修复建筑、建设公会浮空岛，并与其他拓荒者共同对抗虚空 Boss。

### 5.2 用户身份

用户在游戏中被称为：

**校园拓荒者**

用户完成运动后，会获得不同类型的资源：

| 现实行为 | 游戏转化 |
|---|---|
| 完成 3 分钟微运动 | 启动能量 |
| 快走 / 跑步 | 探索值 |
| 健身 / 球类 | 建造值 |
| 拉伸 / 瑜伽 | 恢复值 |
| 鼓励好友 | 羁绊值 |
| 公会协作 | 公会能量 |
| 学院贡献 | 荣耀源力 |

### 5.3 核心资源

| 资源名称 | 作用 |
|---|---|
| XP | 用户等级成长 |
| 源力 | 点亮校园地图、推进主线 |
| 建造值 | 升级公会浮空岛建筑 |
| 羁绊值 | 好友互动、拓荒信号奖励 |
| 公会能量 | 公会战、建筑升级 |
| 荣耀源力 | 学院/班级联赛排名 |

### 8.1 文件结构

推荐第一版使用单文件：

```text
index.html
```

也可以拆分为：

```text
yuedong-planet/
  ├── index.html
  ├── style.css
  ├── script.js
  └── assets/
```

如果为了快速部署和减少路径错误，建议第一版先使用单文件 `index.html`，将 HTML、CSS、JavaScript 写在同一个文件中。

## 11. 三分钟启动舱

### 11.1 模块定位

三分钟启动舱是产品的核心差异化功能，主要面向：

- 拖延用户；
- 低能量日用户；
- ADHD 或注意力困难用户；
- 运动新手；
- 不知道从哪里开始的人。

### 11.2 设计原则

1. 不让用户选择复杂计划；
2. 不要求用户完成高强度运动；
3. 不强调失败；
4. 点击后立即进入倒计时；
5. 完成后立刻奖励；
6. 完成后可选择继续，也可结束。

### 11.3 页面元素

```text
标题：三分钟启动舱
副标题：今天不需要完美运动，先动 3 分钟就算赢。
倒计时显示：03:00
按钮：开始
按钮：暂停
按钮：重置
按钮：演示完成
完成提示：你已经为校园星球注入了第一束源力。
```

### 11.4 完成奖励

完成 3 分钟启动后：

```text
XP +10
源力 +10
建造值 +5
点亮“宿舍营地”
解锁“三分钟勇者”徽章
公会建造值 +5
```

### 11.5 完成后的下一步提示

```text
你已经完成了启动任务。
是否继续一个 5 分钟轻任务？
[继续 5 分钟] [今天到这里也很棒] [换一张随机任务卡]
```

## 12. ADHD 友好设计模块

### 12.1 模块命名

**Focus Move 注意力友好模式**

### 12.2 说明文案

```text
Focus Move 不提供医疗诊断或治疗，仅提供低门槛运动启动、任务拆分和即时反馈支持，帮助用户在注意力分散或执行困难时更容易开始运动。
```

### 12.3 功能清单

| 功能 | 说明 |
|---|---|
| 一键开始 | 点击后直接进入 3 分钟倒计时 |
| 微任务拆分 | 把 20 分钟任务拆成多个小任务 |
| 随机任务卡 | 不知道做什么时直接抽卡 |
| 即时奖励 | 完成后马上给 XP、源力、徽章反馈 |
| 低压力重启 | 中断后重新开始也有奖励 |
| 极简界面 | 启动舱页面减少干扰信息 |
| 可视化进度 | 用进度条显示当前完成度 |
| 正向文案 | 避免“失败”“偷懒”等负面词 |

### 12.4 任务拆分示例

```text
原任务：运动 20 分钟

拆分后：
1. 热身 3 分钟
2. 快走 7 分钟
3. 休息 2 分钟
4. 拉伸 5 分钟
5. 自由活动 3 分钟
```

### 12.5 正向反馈文案

```text
你已经开始了，这比停在原地更重要。
三分钟也算数。
低能量日完成一点点，也是在恢复节奏。
今天不追求完美，只追求启动。
```

## 13. XP、等级与资源规则

### 13.1 XP 规则

| 行为 | XP |
|---|---|
| 完成 3 分钟启动 | +10 |
| 完成普通任务 | +20 |
| 完成挑战任务 | +40 |
| 完成随机任务 | +15 |
| 中断后重启 | +15 |
| 鼓励好友 | +5 |
| 为公会贡献运动 | +10 |

### 13.2 源力规则

| 行为 | 源力 |
|---|---|
| 微任务 | +10 |
| 日常任务 | +20 |
| 跑步任务 | +30 |
| 挑战任务 | +40 |
| 学院联赛贡献 | 同步计入荣耀源力 |

### 13.3 等级规则

```text
等级 = floor(总 XP / 100) + 1
```

示例：

```text
0-99 XP：Lv.1 初醒拓荒者
100-199 XP：Lv.2 星球行者
200-299 XP：Lv.3 校园拓荒者
300-399 XP：Lv.4 源力收集者
400-499 XP：Lv.5 星盟成员
500 XP 以上：Lv.6+ 自定义成长
```

# 21. 学院圣杯联赛

## 21.1 功能定位

学院圣杯是学院/班级层面的集体荣誉系统。用户根据所属学院或班级自动归入阵营，每学期进行排名。

## 21.2 排名原则

不建议直接比较总运动量，因为人数多的学院天然有优势。建议采用综合指标：

```text
学院积分 = 人均源力 × 70% + 活跃率 × 30%
```

其中：

```text
学院人均源力 = 学院总源力 / 活跃人数
学院活跃率 = 本周至少运动一次人数 / 学院总人数
```

第一版可以不真实计算，使用模拟数据展示。

## 21.3 排行榜示例

```text
学院圣杯 · 本学期排名

1. 信息学院：人均源力 860
2. 经管学院：人均源力 790
3. 物理学院：人均源力 735
4. 文学院：人均源力 680

你的学院：物理学院
当前排名：第 3 名
距离上一名还差：55 人均源力
```

## 21.4 奖励设计

| 排名 | 奖励 |
|---|---|
| 第 1 名学院 | 学院圣杯、限定徽章、食堂优惠券 |
| 第 2-3 名 | 限定头像框、打印券 |
| 参与学院 | 公共剧情推进奖励 |
| 个人贡献前 10% | 专属称号 |

## 27. 隐私与安全设计

### 27.1 位置隐私

第一版不使用真实定位。未来如果接入定位，应遵循：

```text
主动开启
用户授权
只显示模糊区域
不显示精确位置
可随时关闭
不默认公开个人轨迹
```

### 27.2 运动数据隐私

默认不展示：

```text
体重
身体照片
精确位置
个人健康敏感数据
```

只展示：

```text
运动次数
运动分钟
源力
建造值
徽章
团队贡献
```

### 27.3 社交压力控制

不建议公开羞辱未完成任务的用户。所有提醒应采用正向表达。

示例：

```text
今天还没启动，要不要先动 3 分钟？
```

避免：

```text
你又失败了。
你拖后腿了。
你断签了。
```

## 31. 产品亮点总结

### 31.1 游戏化亮点

- 足迹占领让校园区域成为轻量竞争场；
- 拓荒信号提供匿名、低压力的运动陪伴；
- 学院圣杯提供集体荣誉和长期参与动机。

- 三分钟启动降低开始门槛；

## 32. 最终产品描述

**跃动星球**是一款面向大学生的游戏化运动激励网页工具。产品将现实运动转化为“源力”“建造值”和“荣耀积分”，用户通过完成微运动、跑步、拉伸、球类和健身任务获得成长奖励，并逐步点亮校园星球地图。产品特别设计了“三分钟启动舱”和 Focus Move 注意力友好模式，帮助拖延、低能量和执行功能困难用户降低运动启动门槛。

在社交层面，跃动星球构建了“校园星盟系统”：用户可以以宿舍、社团或课题组为单位创建公会，建设浮空岛、升级训练场和铁匠铺；也可以参与公会战，对抗世界 Boss；还可以在操场、体育馆、图书馆、宿舍区和湖畔森林进行足迹占领，并通过拓荒信号与附近运动者匿名互动。学院圣杯联赛则将个人运动转化为学院荣誉，增强集体参与感。

第一版产品采用 HTML、CSS 和 JavaScript 实现，使用 localStorage 保存用户进度，并通过 GitHub Pages 或 Netlify 部署为公开访问链接。用户无需安装 App，只需打开网页即可体验完整的运动游戏化流程。
